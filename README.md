# team-5-parking-control
This project is a C++ object-oriented simulation developed to analyze and evaluate access control and staffing strategies for a municipal public parking lot. The purpose of the simulator is to determine how different gate staffing configurations influence congestion levels, wait times, and overall operational efficiency. The parking lot is modeled with a fixed number of parking spaces, two entry gates, and two exit gates, where gates may be staffed or unstaffed depending on the configuration being tested. By simulating realistic vehicle flow and processing behavior, the system provides a controlled environment for comparing alternative management policies.

Vehicles are generated according to a predefined arrival pattern and are assigned a specific parking duration and payment method (cash, debit, or permit). Entry and exit processing times vary depending on the required transaction, and additional delays may occur due to queue length and small random fluctuations to better reflect real-world variability. If the parking lot reaches full capacity, arriving vehicles may either wait for a space to become available or leave immediately, depending on system rules. The simulation therefore incorporates queue management, occupancy tracking, and event-based timing to model the dynamic behavior of the system over time.

The project is organized into three main components: a vehicle arrival generator, a gate staffing configuration generator, and the core simulation engine. The arrival generator produces a standardized input file listing vehicles in order of arrival, along with their parking duration and payment method. The staffing configuration generator specifies when each gate is staffed or unstaffed. The simulation engine then reads both input files and executes a discrete-event simulation to model vehicle movement, gate processing, and parking occupancy. Throughout execution, the program records performance metrics such as average wait times at entry and exit gates, gate utilization and idle time, the number of vehicles turned away, and overall parking space utilization.

By maintaining the same vehicle arrival pattern across multiple simulations, this project allows for direct comparison between different staffing strategies. The results can be used to evaluate whether proposed policy changes reduce congestion, improve customer experience, and optimize resource allocation. Overall, the simulator demonstrates the application of object-oriented programming principles, file input/output handling, queue-based modeling, and performance analysis within a realistic systems engineering context.
